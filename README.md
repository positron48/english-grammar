# Grammar Course (EN) — интерактивный курс на JSON

Этот репозиторий содержит “движок” интерактивного курса английской грамматики на чистом `HTML/CSS/JavaScript`, где контент каждой темы хранится в виде сгенерированных JSON-файлов.

Для пользователя:
- вы учите теорию прямо на странице;
- после каждого теоретического блока отвечаете на inline-квизы;
- финальная страница собирает рандомизированный тест.

Для автора/сборщика контента:
- есть админ-панель для просмотра всех глав;
- есть скрипты для сборки `05-final.json` и валидации структуры/логики вопросов.

## Что внутри

- `test/` — сайт для обучения (главы + inline-квизы + итоговый тест).
- `admin/` — админ-панель просмотра/валидации глав и вопросов.
- `chapters/` — данные курса (по одной папке на главу, в каждой есть `05-final.json`).
- `scripts/` — скрипты сборки и валидации JSON (например, сборка `05-final.json`).
- `config/` и `prompts/` — вспомогательные конфиги/шаблоны для генерации контента (LLM-процесс).
- `Makefile` — удобные команды “собрать/проверить/запустить”.

## Быстрый старт (без сборки)

Данные курса уже лежат в репозитории (например, `chapters/*/05-final.json`), поэтому для обычного просмотра “сборка” не нужна — достаточно поднять локальный HTTP-сервер.

1. Запустите сервер из корня проекта:
```bash
python3 -m http.server 8001
```
2. Откройте:
```text
http://localhost:8001/test/
```

Админ-панель:
- откройте `http://localhost:8001/admin/`
- либо используйте `make admin` (см. ниже).

Важно: “просто открыть файл” в браузере недостаточно, потому что `fetch` к локальным JSON упирается в CORS.

## Запуск через Makefile

Из корня проекта:

- `make test` — старт тестовой системы: `http://localhost:8001/test/`
- `make admin` — старт админ-панели: `http://localhost:8000/admin/`
- `make dev` — оба сервера сразу

Эти команды также обновляют индексы глав перед запуском.

## Запуск через Docker

Можно собрать контейнер и запустить сайт:
```bash
docker build -t english-grammar:local .
docker run -p 8080:80 english-grammar:local
```

Откройте:
```text
http://localhost:8080/test/
```

Примечание про админку в Docker: в `nginx.conf` `/admin/` защищён HTTP Basic Auth. В репозитории нет файла с учётными данными (`/etc/nginx/.htpasswd`), поэтому для полноценной админки в контейнере может понадобиться доп. настройка. Для просмотра/редактирования данных проще использовать `make admin`.

## Как “собрать” данные курса (опционально)

Если вы меняете промежуточные JSON в `chapters/*` (outline/questions/theory blocks), то нужно пересобрать:

1. Сборка `05-final.json` для всех глав:
```bash
make final
```

2. Валидация:
```bash
make validate-all
```

3. Обновление индексов:
- для админки: `make update-admin-index`
- для тестовой системы: `make update-test-index`

Технически сборка `05-final.json` делается скриптом `scripts/assemble-chapter.sh`, который:
- объединяет `01-outline.json` + `02-theory-blocks/*.json` + `03-questions.json`;
- сохраняет результат в `05-final.json`;
- (опционально) проверяет соответствие схеме через `ajv-cli`, если он установлен.

## Зависимости (если хотите пересобирать данные)

Для пересборки и валидации JSON обычно нужны:
- `node` (генерация индексов глав: `admin/generate-index.js`, `test/scripts/generate-chapters-index.js`);
- `python3` (скрипты валидации внутри `scripts/`);
- `jq` (используется в `scripts/assemble-chapter.sh`).

`ajv-cli` — опционально (используется только если установлен).

## Структура файлов

### Верхний уровень
```text
.
├── admin/                  # админ-панель
├── test/                   # сайт для обучения
├── chapters/              # данные курса (контент)
├── scripts/               # сборка/валидация данных
├── config/                # конфиги генерации (status/config/templates)
├── prompts/               # промпты для генерации контента
├── Makefile               # команды “собрать/проверить/запустить”
├── nginx.conf             # nginx в Docker
├── Dockerfile             # сборка контейнера
└── 02-chapter-schema.json # JSON-схема одной главы
```

### Одна глава (`chapters/<chapter_id>/`)
Внутри папки главы обычно лежат:

```text
chapters/<chapter_id>/
├── 01-outline.json        # план/метаданные (chapter_outline)
├── 02-theory-blocks/      # теория, разбитая на блоки (несколько json)
├── 03-questions.json      # банк вопросов (question_bank)
├── 05-final.json          # финальная глава, которую читает `test/`
└── 05-validation.json     # результат валидации (который показывает `admin/`)
```

UI-часть:
- `test/` ожидает `05-final.json`;
- `admin/` дополнительно показывает `01-outline.json` и читает `05-validation.json`.

Индексы глав:
- `admin/data/chapters-index.json` (генерируется `admin/generate-index.js`)
- `test/data/chapters-index.json` (генерируется `test/scripts/generate-chapters-index.js`)

## Схема данных главы

Формат `05-final.json` описан в `02-chapter-schema.json`.

Если вы хотите расширять типы вопросов или правила блоков — начните со схемы и скриптов валидации (`scripts/validate-chapter.sh`).
